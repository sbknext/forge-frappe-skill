Run a focused sprint retrospective that produces 1-3 actionable improvements, not just discussion.

## When to use
End of every sprint or major project milestone. 30 minutes is enough if the format is tight.

## Agenda
```
0–5 min:   Set the stage — one word each about how the sprint felt
5–15 min:  What went well (write on sticky notes, cluster)
15–25 min: What to improve (write, cluster, dot-vote top 3)
25–30 min: Actions — for each top item, write who does what by when
```

## Template
```markdown
## Sprint [N] Retro — [Date]
**Facilitator**: 
**Team**: 

### What went well
- 
- 

### What to improve (top-voted)
1. [Issue] — votes: ___
2. [Issue] — votes: ___
3. [Issue] — votes: ___

### Actions (MUST have owner + date)
| Improvement | Action | Owner | By when |
|-------------|--------|-------|---------|
| | | | |

### Check-in on last retro's actions
| Last action | Done? | Notes |
|-------------|-------|-------|
| | | |
```

## Tips
- The action section is the only output that matters. Discussion without action is venting, not retro.
- Limit to 3 improvements max. Teams that identify 10 improvements fix 0.
- Start by checking last retro's actions. If they were never done, understand why before generating new ones.
- For remote teams: Miro or FigJam work well for sticky notes; a shared doc works too.
