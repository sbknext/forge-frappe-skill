A pre-flight checklist for starting a new project so nothing critical is missing at week one.

## When to use
Before any project that involves >2 people and >2 weeks of work.

## Checklist
```markdown
### Before kickoff meeting
- [ ] Problem statement written (1 paragraph — what problem, for whom, why now)
- [ ] Success metrics defined (what does done look like, measurably)
- [ ] Scope boundary drawn (what is OUT of scope, explicitly)
- [ ] Stakeholders identified (RACI: Responsible, Accountable, Consulted, Informed)
- [ ] Timeline skeleton drafted (milestones, not detailed tasks)
- [ ] Known risks listed (at least 3)

### At kickoff meeting
- [ ] Problem statement agreed (not just acknowledged)
- [ ] Success metrics agreed
- [ ] Scope agreed — especially what's out
- [ ] Roles confirmed (who does what)
- [ ] Communication cadence set (standup? weekly sync? async-only?)
- [ ] Where decisions get logged agreed
- [ ] First milestone date set

### Week 1
- [ ] Project space created (Jira/Linear/Notion/Slack channel)
- [ ] All stakeholders have access
- [ ] First set of tasks broken down
- [ ] Blockers/dependencies identified and flagged
- [ ] First check-in scheduled
```

## Tips
- The "out of scope" list is more important than the "in scope" list. Scope creep always comes through undefined edges.
- RACI: if more than one person is Accountable for a deliverable, no one is. One A per item.
- Don't skip risk listing. Three risks in 5 minutes is enough — it forces thinking about what can go wrong.
